# LINE友だち追加を完了させるための6セクションLP実行計画書

## 1. 実行計画書（AI理解向け・構造化版）

### 1-1. 目的
`line-coupon` ページを、説明中心ページではなく、**LINE友だち追加を完了させるための専用LP**に再設計する。  
このページの最優先コンバージョンは **「友だち追加ボタン押下」** とする。  
ID連携やECアカウントの深い説明は主目的から外し、初回導線では扱いを軽くする。

### 1-2. ページの役割
このページの役割は次の4点に限定する。

1. LINE会員登録のメリットを短時間で理解させる  
2. 友だち追加を完了させる  
3. クーポン利用の流れを不安なく理解させる  
4. 商品一覧ページへ送客する  

### 1-3. 非目的
このページでは次を主目的にしない。

- ID連携の詳細説明
- ECアカウント登録の詳細説明
- メール登録施策との同時訴求
- 情報量の多い比較説明
- 長い手順説明

### 1-4. 設計原則
- 1ページ1目的で設計する  
- ファーストビューでCTAを見せる  
- 情報は削ってもCTAは削らない  
- ボタン優先、QRは補助  
- 説明より先に利益を提示する  
- 「LINE会員登録」という言葉でまとめ、技術用語は出しすぎない  
- ブランドトーンは上品・整理・過度な装飾なしとする  

### 1-5. KPI
一次KPI  
- 友だち追加ボタンのクリック  
- QR経由登録の発生

二次KPI  
- クーポンリンク遷移  
- 商品一覧遷移  
- その後の購入

### 1-6. ページ構成
ページは次の6セクションで構成する。

#### セクション1 ヒーロー
目的：ページ訪問直後に「何が得で、何を押せばよいか」を伝える。

表示要素  
- 見出し
- サブコピー
- 主CTAボタン
- 小さな補足文

推奨文言  
- 見出し：LINE会員登録で500円クーポンプレゼント
- サブコピー：新作のお知らせや再入荷情報もLINEでお届けします。
- ボタン：LINE友だち追加はこちら
- 補足：登録後、クーポンのご案内をLINEでお送りします。

成功条件  
- ファーストビューでボタンが見える
- 3秒以内にページの目的が伝わる

#### セクション2 特典3つ
目的：登録の価値を短く具体化する。

表示要素  
- セクション見出し
- 特典カード3枚

推奨特典  
1. 500円クーポン  
2. 新作情報をすばやくチェック  
3. 再入荷情報を見逃しにくい  

成功条件  
- 読み込み負荷が低い
- 特典は3つ以内
- 値引きだけでなく情報価値も伝わる

#### セクション3 友だち追加エリア
目的：実際に登録を完了させる中心ブロック。

表示要素  
- 見出し
- 説明文
- 友だち追加ボタン
- QRコード
- 補足文

仕様  
- スマホ：ボタン優先
- PC：QR補助
- 使用リンク：EC_LINE500の友だち追加経路
- 使用QR：EC_LINE500用QR画像

成功条件  
- ページ内で最も押しやすい
- ボタンとQRの役割が明確
- 登録無料の安心感がある

#### セクション4 かんたん3ステップ
目的：利用イメージを短く見せて不安を下げる。

表示要素  
- 見出し
- ステップ1〜3

推奨ステップ  
1. LINEで友だち追加  
2. クーポンを受け取る  
3. お買い物を楽しむ  

成功条件  
- 長文手順の代替になっている
- 1ステップ1メッセージで理解できる

#### セクション5 安心材料
目的：登録前の不安を解消する。

表示要素  
- 見出し
- 3〜4個の短いQ&A

推奨Q&A  
- 登録に料金はかかりますか？ → 無料です  
- クーポンはどこで使えますか？ → トップシェルECで使えます  
- 配信が多すぎませんか？ → 必要な情報を中心にお届けします  
- やめたくなったら？ → 通知設定見直しやブロック可能です  

成功条件  
- 不安を減らす
- 長いFAQにしない

#### セクション6 商品一覧導線
目的：登録後の次行動を明確にする。

表示要素  
- 見出し
- 商品一覧ボタン
- 補足文

推奨文言  
- 見出し：クーポンを使ってお買い物へ
- ボタン：商品一覧を見る
- 補足：登録後に届くクーポンをご利用ください。

成功条件  
- LPがそこで終わらず買い物導線へつながる
- `collections/all` へ自然に遷移できる

### 1-7. 削除・後回し対象
初回改善では次を削る、または後段へ移す。

- ID連携の詳細説明
- 会員登録済み / 初めて の複雑な分岐タブ
- メール登録施策
- 長い図解
- 冒頭の長文説明
- CTAより上の情報量過多

### 1-8. 実装優先順位
1. ヒーロー差し替え  
2. 特典3つ整理  
3. 友だち追加エリア最適化  
4. 3ステップ簡略化  
5. FAQ短文化  
6. 商品一覧導線追加  

### 1-9. 完了判定
このLPは、以下を満たせば第一段階完了とする。

- ファーストビューでCTAが見える
- LINE登録メリットが3つ以内で整理されている
- 友だち追加ボタンとQRが正しく配置されている
- 商品一覧への導線がある
- ID連携説明が主役になっていない
