# 一週間ぶりの再開＆バージョンアップ後の手順

一週間ほど作業を中断していたとき、またその間に「バージョンアップ」があったときに、スムーズに再開するための手順です。エンジニアでなくても実行できる内容にしています。

---

## 1. 再開前にやること（5分）

### 1-1. フォルダを開く
- このテーマのフォルダを開く  
  `theme_export__shop-topshell-jp-dawn__02MAR2026-0544pm`
- Cursor で「ファイル → フォルダを開く」から同じフォルダを開いておく

### 1-2. 変更の有無を確認する（Git）
- 左サイドバーの **ソース管理（枝マーク）** をクリック
- 「変更」にファイルがたくさん出ている場合は、**前回の作業内容がまだコミットされていない**状態です
- この段階では **何もコミットしなくて大丈夫** です。「変更がある」ことだけ確認できればOK

### 1-3. 前回どこまでやったか思い出す
- `docs/` フォルダ内の **開店作業_チェックリスト.md** や **開店作業_操作手順とチェック内容.md** を開く
- チェックが付いている項目＝完了、付いていない＝これからやる、と判断できます
- 会話ログやメモがあれば「最後に何をしたか」を軽く見ておく

---

## 2. テーマを安全に起動する（再開のメイン作業）

### 2-1. ターミナルを開く
- Cursor で **ターミナル → 新しいターミナル**（または `` Ctrl+` ``）
- 開いたターミナルの **今いる場所** が、このテーマのフォルダになっているか確認  
  （パスに `theme_export__shop-topshell-jp-dawn__02MAR2026-0544pm` が含まれていればOK）

### 2-2. ストアを指定してプレビューを起動
次のコマンドを **1行そのまま** 入力して Enter：

```bash
shopify theme dev --store=5aj9ns-tu.myshopify.com
```

- 初回や長期間あいだを空けたあとは、**ログイン** を求められることがあります
- 画面の指示に従い、表示された **認証コード** をブラウザで入力
- 成功すると **プレビュー用のURL**（例: `http://127.0.0.1:9292`）が表示されます

### 2-3. ブラウザでプレビューを開く
- 表示されたURLをブラウザで開く（または Cursor が「開く」リンクを出していればそれをクリック）
- トップページや、前回編集していたページが表示されれば **再開完了** です

---

## 3. バージョンアップがあったときの考え方

「バージョンアップ」は主に次の2つです。

| 何が | 説明 | やること |
|------|------|----------|
| **Shopify CLI の更新** | パソコンに入っている `shopify` コマンドのバージョン | 必要なら更新（下記 4） |
| **テーマや Shopify 側の更新** | ストアやテーマの仕様変更 | 通常は **何もしない** でOK |

- **テーマのコード（Liquid / CSS など）は、基本的に「保存したまま」** で問題ありません
- 再開時は **まず 2 の「テーマを起動する」だけ** やって、プレビューが開けばそのまま作業を再開して大丈夫です

---

## 4. Shopify CLI を更新する場合（任意）

「`shopify theme dev` が動かない」「エラーで起動できない」といったときだけ、次を試します。

### 4-1. Homebrew で更新（推奨）
ターミナルで次を **順番に** 実行：

```bash
brew update
brew upgrade shopify-cli
```

- 完了したら、もう一度 **2-2** の `shopify theme dev --store=5aj9ns-tu.myshopify.com` を実行

### 4-2. バージョン確認（やりたければ）
```bash
shopify version
```
- 数字が表示されれば CLI は入っています。細かいバージョン番号は気にしなくてOKです

---

## 5. 作業を再開したあと（本番に反映したいとき）

- プレビューで見た目を確認し、問題なければ **本番ストアに反映** します
- 手順は **開店作業_操作手順とチェック内容.md** の「テーマのアップロード」を参照
- 要約すると：
  1. ターミナルで `shopify theme push --store=5aj9ns-tu.myshopify.com`
  2. 管理画面でそのテーマを「公開」

---

## 6. 困ったときの確認リスト

| 症状 | やること |
|------|----------|
| `shopify` コマンドがない | **4. Shopify CLI を更新する** の前に、`brew install shopify-cli` を実行（.cursor/rules の通り） |
| ログインを求められる | 表示された認証コードをブラウザで入力する |
| ストアが違う・見つからない | `--store=5aj9ns-tu.myshopify.com` を付けて `shopify theme dev` を実行しているか確認 |
| プレビューが真っ白・おかしい | ブラウザのキャッシュを消す、またはシークレットウィンドウで同じURLを開いてみる |
| ファイルを触ったか不安 | 左サイドバーの「ソース管理」で変更一覧を確認 |

---

## まとめ：再開時の最小手順

1. **テーマのフォルダを Cursor で開く**
2. **ターミナルで**  
   `shopify theme dev --store=5aj9ns-tu.myshopify.com`  
   **を実行**
3. **表示されたURLをブラウザで開いてプレビューを確認**
4. 問題なければ **いつも通り編集やチェックリストの続き** を進める  
5. 本番に反映したくなったら **開店作業_操作手順** の「テーマのアップロード」を行う  

バージョンアップ後も、**まずはこの「起動してプレビュー」ができていれば、そのまま作業を再開して大丈夫**です。
