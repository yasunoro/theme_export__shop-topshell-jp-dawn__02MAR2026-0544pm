# トップシェルEC 開店作業 — 計画用プロンプト

以下のプロンプトをそのままチャットに貼り、開店作業の全体計画と手順を一度に整理してください。

---

## コピー用（ここから）

```
トップシェルEC を本日（または指定日）に Shopify で開店します。問題なくスムーズに回転するよう、開店前・開店時・開店後にやることを「実行順」で整理したチェックリスト形式の計画を出してください。

次の観点をすべて含めてください。各項目は「何を・どこで・どう確認するか」が分かる具体的な手順にし、必要なら Shopify 管理画面のパス（例: 設定 > ドメイン）も書いてください。

1. **ドメイン・DNS**
   - 使用するドメイン（独自ドメイン or myshopify.com）
   - Shopify 側のドメイン設定手順
   - レジストラ／DNS の A レコード・CNAME の設定値と手順
   - SSL（https）の確認方法
   - リダイレクト（www → 非 www など）の設定有無

2. **決済・送料・税金**
   - 決済手段の有効化とテスト
   - 送料・配送設定の最終確認
   - 税設定（軽減税率など）の確認

3. **計測・分析（GA4 など）**
   - Google Analytics 4（GA4）のプロパティ作成有無と計測 ID の設定場所
   - Shopify への GA4 タグ設置方法（Google タグマネージャ or 直接）
   - コンバージョン・購入イベントの設定
   - その他（Search Console、広告タグなど）があれば記載

4. **ストアフロント・表示**
   - テーマの本番反映とプレビュー確認
   - 重要ページ（トップ・商品一覧・カート・注文完了・特定商取引・プライバシーポリシー）の内容確認
   - 在庫・価格・表示／非表示の最終チェック

5. **通知・顧客管理**
   - 注文確認・発送通知メールの文面・送信元の確認
   - 顧客アカウント・ログインの要否と設定

6. **既存 Square EC の閉店**
   - 閉店日と告知のタイミング
   - 在庫・注文の打ち切り手順
   - ドメイン・決済の停止手順
   - 閉店後のリダイレクト（Square の URL → 新 Shopify の URL）の有無と設定

7. **その他・開店後**
   - 開店直後に確認する項目（注文受信、メール送信、計測の動作）
   - 問い合わせ窓口・サポート体制の確認

出力形式は、番号付きの大項目と、その下に「□ 〇〇をする」のようなチェック可能な小項目でお願いします。本日開店を前提に、今日中に必ずやる項目と、開店後や後日でよい項目を分けて書いてもらえると助かります。
```

## 使い方

1. 上記「ここから」～「ここまで」のブロックをコピーする。
2. Cursor のチャット（または ChatGPT 等）に貼り付けて送信する。
3. 返ってきた計画を、このリポジトリの `docs/` や Notion・スプレッドシートなどに保存し、実行時にチェックしていく。

必要に応じて、プロンプト内の「本日」「トップシェルEC」「Square」などを実際の店名・日程に書き換えて使ってください。
