# 開店作業 — 操作手順とチェックする具体的な内容

チェックリストに沿い、**操作手順** と **チェックする具体的な内容** を項目ごとに記載。この順で進めてください。

---

## 2. 決済・送料・税金

### 2-1. 決済手段の有効化確認

**操作手順**
1. Shopify 管理画面にログインする
2. 左メニュー **設定**（歯車アイコン）をクリック
3. **支払い** をクリック
4. 画面をスクロールし、「支払いプロバイダー」一覧を確認する

**チェックする内容**
- [ ] 利用予定の決済（例: Shopify Payments / クレジットカード、Shop Pay、代引き、コンビニ等）が一覧に表示されているか
- [ ] その決済のステータスが「有効」または「アクティブ」になっているか（無効・未設定のものは開店後に選べない）
- [ ] テストモードを使う場合：「テスト」または「Bogus Gateway」が有効か確認

---

### 2-2. テスト注文で決済確認

**操作手順**
1. ストアのフロント（プレビュー or 本番URL）を開く
2. 商品を 1 点カートに追加する
3. **購入手続きに進む**（チェックアウト）をクリック
4. テスト用の配送先・支払い情報を入力する  
   - **テストモード時:** カード番号は Shopify のテスト用番号（例: 4242 4242 4242 4242）を使用
5. 注文を完了する

**チェックする内容**
- [ ] 決済エラーが出ずに注文が完了するか
- [ ] 管理画面 **注文** にその注文が表示され、支払い状況が「支払い済み」等になっているか
- [ ] 本番決済を使う場合は、少額（例: 1円）で実際に決済が通るか

---

### 2-3. 送料・配送の最終確認

**操作手順**
1. 管理画面 **設定** → **配送と発送** を開く
2. 「配送ゾーンと料金」または「配送プロファイル」を確認する
3. 各ゾーン（国・地域）の配送方法と料金を確認・必要なら編集する

**チェックする内容**
- [ ] 販売する国・地域（例: 日本）が配送ゾーンに含まれているか
- [ ] 送料（無料 / 一律 / 重量別など）が開店仕様どおりか
- [ ] 配送方法（宅配・コンビニ・店舗受け取り等）の名前と料金が正しいか
- [ ] チェックアウト画面で「配送」が選べる状態になるか（プレビューで確認推奨）

---

### 2-4. 税設定の確認

**操作手順**
1. 管理画面 **設定** → **税金** を開く
2. 「国・地域の税金」「商品の税金」などの項目を確認する

**チェックする内容**
- [ ] 日本向け: 消費税（標準 10%・軽減 8%）の設定が意図どおりか
- [ ] 価格表示が「税込」か「税別」か、ストア方針と一致しているか
- [ ] 該当する国・地域に税率が設定されているか

---

## 3. 計測・分析（GA4 など）

### 3-1. GA4 プロパティと測定 ID

**操作手順**
1. [Google アナリティクス](https://analytics.google.com/) にログイン
2. **管理**（左下の歯車）→ **データストリーム** を開く
3. 該当するウェブのデータストリームを選択（または新規作成）
4. 「ストリームの詳細」内の **測定 ID**（G-XXXXXXXXXX）をコピーする

**チェックする内容**
- [ ] GA4 プロパティが作成済みか
- [ ] 測定 ID をメモまたは保存しているか（タグ設置時に必要）

---

### 3-2. Shopify に GA4 タグを設置

**操作手順（方法 A: GTM の場合）**
1. [Google タグマネージャ](https://tagmanager.google.com/) でコンテナを作成
2. GA4 設定タグを追加し、上記の測定 ID を入力
3. Shopify 管理画面 **設定** → **チェックアウトのカスタマイズ** を開く
4. 「追加のスクリプト」または「ヘッダー／フッター」に GTM のスニペット（`<script>...</script>`）を貼る  
   - またはテーマ編集で **layout/theme.liquid** の `<head>` 内に貼る

**操作手順（方法 B: gtag 直接の場合）**
1. テーマの **layout/theme.liquid** を開く（オンラインストア > テーマ > 編集 > レイアウト）
2. `<head>` 内のなるべく上側に、GA4 の gtag.js スクリプトを貼る（測定 ID を G-XXXXXXXXXX の部分に代入）

**チェックする内容**
- [ ] ストアのトップページを開いた状態で、ブラウザの開発者ツール「ネットワーク」または「Elements」で gtag や GTM が読み込まれているか
- [ ] GA4 の「リアルタイム」で、ストアにアクセスするとユーザーが 1 人表示されるか

---

### 3-3. 購入イベント（purchase）の確認

**操作手順**
1. 上記のタグ設置後、テスト注文を 1 件完了させる
2. GA4 **レポート** → **リアルタイム** または **管理** → **イベント** を開く
3. 数分待ってから「purchase」イベントが記録されているか確認する

**チェックする内容**
- [ ] イベント一覧に `purchase` が表示されるか
- [ ] パラメータに金額（value）などが入っているか（あれば）

---

### 3-4. Search Console・広告タグ（あれば）

**操作手順**
- **Search Console:** プロパティを追加 → サイトマップに `https://あなたのドメイン/sitemap.xml` を送信
- **広告タグ:** Meta ピクセルや Google 広告タグを、GTM または theme.liquid の該当位置に設置

**チェックする内容**
- [ ] Search Console でサイトマップが認識されているか
- [ ] 広告タグが発火しているか（各ツールのデバッグ機能やタグアシスタントで確認）

---

## 4. ストアフロント・表示

### 4-1. テーマの公開状態

**操作手順**
1. 管理画面 **オンラインストア** → **テーマ** を開く
2. 使用するテーマの「•••」メニューから **公開** を選択（まだの場合）

**チェックする内容**
- [ ] 本番で表示したいテーマに「現在のテーマ」または「公開」と表示されているか
- [ ] プレビューでストアが想定どおりのデザインで表示されるか

---

### 4-2. 重要ページの内容確認

**操作手順**
1. ストアのプレビューまたは本番 URL を開く
2. 次のページを順に開き、内容を確認する

**チェックする内容**
- [ ] **トップページ:** 告知・商品・ナビゲーションが正しく表示されるか
- [ ] **商品一覧（コレクション）:** 商品が表示され、リンクが切れていないか
- [ ] **商品詳細:** 価格・在庫・カートに追加が動作するか
- [ ] **カート・チェックアウト:** 送料・税・決済まで進めるか
- [ ] **注文完了（サンクスページ）:** 注文番号やメッセージが表示されるか
- [ ] **特定商取引法に基づく表記:** ページが存在し、必要事項が記載されているか
- [ ] **プライバシーポリシー:** ページが存在し、リンクが有効か

---

### 4-3. 在庫・価格・販売状態の最終確認

**操作手順**
1. 管理画面 **商品** → **商品** を開く
2. 一覧または各商品の編集画面で確認する

**チェックする内容**
- [ ] 販売したい商品の「販売する」がオンになっているか
- [ ] 在庫数が想定どおりか（在庫管理している場合）
- [ ] 価格・比較価格（セール価格）が正しいか
- [ ] 非表示にしたい商品は「販売する」オフ or ドラフトになっているか

---

## 5. 通知・顧客管理

### 5-1. 注文確認・発送通知メールの確認

**操作手順**
1. 管理画面 **設定** → **通知** を開く
2. 「顧客通知」の **注文確認** をクリックして内容を確認・編集
3. 同様に **発送通知**（配送通知）を確認・編集する

**チェックする内容**
- [ ] 文面が自社のトーン・必要事項（注文番号、金額、問い合わせ先など）を満たしているか
- [ ] 送信元名・メールアドレスが正しいか
- [ ] 署名や会社名が入っているか（必要なら編集）

---

### 5-2. 注文確認メールの受信テスト

**操作手順**
1. テスト注文を 1 件完了する（自分が受け取れるメールアドレスで）
2. メール受信トレイ（迷惑フォルダ含む）を確認する

**チェックする内容**
- [ ] 注文確認メールが届いているか
- [ ] 内容（注文番号・商品・金額）が正しいか
- [ ] リンクやレイアウトが崩れていないか

---

### 5-3. 顧客アカウント設定

**操作手順**
1. 管理画面 **設定** → **チェックアウト** を開く
2. 「顧客アカウント」の項目を確認する（必須 / 任意 / 不要 など）
3. 必要なら「アカウントの作成」を「任意」または「不要」に変更し保存

**チェックする内容**
- [ ] ゲスト購入のみでよいか、会員登録を促すかが方針と一致しているか
- [ ] ログイン・会員登録のリンクがストア上で正しく動くか（プレビューで確認）

---

## 6. 既存 Square EC の閉店（該当する場合）

### 6-1. 閉店告知

**操作手順**
- 自社サイト・SNS 等で「〇月〇日をもって Square での販売を終了し、〇〇（Shopify の URL）に移転します」と事前に告知する

**チェックする内容**
- [ ] 閉店日が明確に伝わっているか
- [ ] 新しいストアの URL が正しく記載されているか

---

### 6-2. 注文・在庫の打ち切り

**操作手順**
- Square の管理画面で、新規注文の受付を止める、在庫を切る、または受付終了の設定を行う

**チェックする内容**
- [ ] 閉店日以降、Square で新規注文が入らない状態になっているか

---

### 6-3. ドメイン・決済の停止

**操作手順**
- Square で使用していたドメイン・決済サービスについて、契約に従って停止手続きを行う

**チェックする内容**
- [ ] 支払い・ドメインの解約・停止が手続き済みか

---

### 6-4. リダイレクト（任意）

**操作手順**
- 旧 Square の URL から新 Shopify の URL へ 301 リダイレクトを設定する（レジストラの DNS 設定や Square の設定画面で実施）

**チェックする内容**
- [ ] 旧 URL にアクセスすると新ストアの URL に転送されるか

---

## 開店時（本日）

### 開店時-1. パスワード保護の解除

**操作手順**
1. 管理画面 **オンラインストア** → **テーマ** を開く
2. 現在のテーマの **•••** → **パスワードを無効にする** または設定から「ストアのパスワード」をオフにする  
   - 場合によっては **設定** → **一般** の「ストアのパスワード」を無効にする

**チェックする内容**
- [ ] ストアの URL をシークレットウィンドウで開いたとき、パスワード画面が出ずにトップページが表示されるか

---

### 開店時-2. 公開状態の再確認

**操作手順**
- **オンラインストア** → **テーマ** で「現在のテーマ」が意図したテーマか確認する

**チェックする内容**
- [ ] 本番で見せたいテーマが公開されているか

---

### 開店時-3. 一般客目線での表示確認

**操作手順**
- ストアのトップページ・商品ページをブラウザで開き、一般客がアクセスしたときの見え方を確認する

**チェックする内容**
- [ ] トップ・商品が問題なく表示されるか
- [ ] カート・購入手続きへ進めるか

---

## 開店後（本日〜数日以内）

### 7-1. テスト注文の受信確認

**操作手順**
1. 開店後にテスト注文を 1 件入れる
2. 管理画面 **注文** でその注文が表示されるか確認する

**チェックする内容**
- [ ] 注文が一覧に表示されるか
- [ ] 支払い状況・配送先が正しいか

---

### 7-2. 注文確認メールの確認

**操作手順**
- 上記テスト注文の注文確認メールが届いているか確認する

**チェックする内容**
- [ ] メールが届くか
- [ ] 内容が正しいか

---

### 7-3. GA4 の計測確認

**操作手順**
- GA4 の **リアルタイム** で、ストアにアクセスしたときのページビュー・イベント（purchase 等）を確認する

**チェックする内容**
- [ ] アクセスが計測されているか
- [ ] テスト購入で purchase が記録されているか

---

### 7-4. 問い合わせ窓口の確認

**操作手順**
- ストアに記載している問い合わせ先（メール・フォーム・電話）に、テストで問い合わせを送る

**チェックする内容**
- [ ] 問い合わせが届くか
- [ ] ストア上の表記（アドレス・電話番号）が正しいか
- [ ] サポート体制（対応時間・返信方針）がチーム内で共有されているか

---

各項目を実施したら、メインの **開店作業_チェックリスト.md** の該当する □ にチェックを入れて進めてください。不明点があればその項目を指定して質問してください。
